Pour cette mission, vous disposez déjà de la solution. Le seul problème, c'est la présence d'un bug. Il vous faut le trouver et le réparer. L'usage de "import", "exec" et "eval" est interdit pour cette mission.

Une chaîne de caractères vous est donnée ; elle contient une liste d'entiers, ou de sous-listes. Les entiers peuvent être précédés des signes plus ou moins. Si la chaîne (string) donnée ne contient aucun array, alors lever ValueError. Pour une chaîne (string) incorrectement formatée, lever ValueError. Le éléments du array sont séparés par des virgules.

Input: Une chaîne de caractères (string).

Output: La liste de sous-listes ou d'entiers.

Exemple: 

parse_array("[1, 2, 3]") == [1, 2, 3]
parse_array("[[1], 2, 3]") == [[1], 2, 3]
parse_array("[-3, [-2, 0], 10]") == [-3, [-2, 0], 10]

Application: Cette mission a été pensée pour évaluer vos capacités à trouver des bugs. Le code parser lui-même peut être modifié et amélioré pour fonctionner comme une partie d'un système de parsing plus complexe.

Préconditions:
profondeur < 5
∀ x ∈ data : -1000 < x < 1000
